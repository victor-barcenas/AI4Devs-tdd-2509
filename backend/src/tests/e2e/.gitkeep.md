# End-to-End Tests

This directory contains end-to-end tests that verify complete user workflows.

## Structure
- Test files should follow the naming convention: `{feature}.e2e.test.ts`
- Tests should simulate real user interactions
- Use browser automation tools (Playwright/Cypress)

## Example Test Structure
```typescript
describe('Candidate Registration E2E', () => {
  it('should allow user to register new candidate', async () => {
    // Navigate to registration page
    // Fill out form
    // Submit form
    // Verify success message
    // Verify data in database
  });
});
```

## Running E2E Tests
```bash
npm run test:e2e
```
