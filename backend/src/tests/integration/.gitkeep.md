# Integration Tests

This directory contains integration tests that verify the interaction between different modules and components.

## Structure
- Test files should follow the naming convention: `{module}.integration.test.ts`
- Tests should verify component interactions
- Use real database connections with test data

## Example Test Structure
```typescript
describe('Candidate Integration Tests', () => {
  beforeAll(async () => {
    // Setup test database
  });
  
  afterAll(async () => {
    // Cleanup test database
  });
  
  it('should create candidate with all related data', async () => {
    // Test full candidate creation flow
  });
});
```

## Running Integration Tests
```bash
npm test -- --testPathPattern=integration
```
