# Unit Tests

This directory contains unit tests for individual components, functions, and classes.

## Structure
- Test files should follow the naming convention: `{module}.test.ts`
- Each test file should test a single module or component
- Use the AAA pattern: Arrange, Act, Assert

## Example Test Structure
```typescript
describe('CandidateService', () => {
  describe('addCandidate', () => {
    it('should create candidate with valid data', async () => {
      // Arrange
      const validData = { /* test data */ };
      
      // Act  
      const result = await addCandidate(validData);
      
      // Assert
      expect(result).toBeDefined();
    });
  });
});
```

## Running Unit Tests
```bash
npm test -- --testPathPattern=unit
```
